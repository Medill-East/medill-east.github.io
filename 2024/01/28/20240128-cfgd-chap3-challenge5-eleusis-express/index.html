该篇是《Challenges for Game Designers》第三章的第 5 个挑战—— New Eleusis。

挑战介绍

题目

原文中提到的是 [[New Eleusis]]，后来稍微了解了一下这个游戏的历史：

  • 1956，[[Robert Abbott]] 发明了这个全新的纸牌游戏 [[Eleusis]]，[[Martin Gardner]] 在 1959 年的 [[Scientific American]] 中介绍了 [[Eleusis]]
  • 1973 年，[[Robert Abbott]] 在原游戏基础上对其进行了改进，称其为 [[New Eleusis]]（虽然他后来将“New”废除了，因为该游戏成为了后续时间的标准版本，但这里还是沿用这个说法以便区分）。[[Martin Gardner]] 在 1977 年 10 月的 [[Scientific American]] 中刊登了 [[New Eleusis]]
  • 2006 年，[[John Golden]] 继续对游戏进行了改进，[[Robert Abbott]] 将这个版本称为 [[Eleusis Express]]，这也是接下来会介绍的版本

接下来是对 Eleusis Express 页面的翻译：

  • 概述
    • 一个玩家需要设计一条关于牌应该如何被打出的秘密规则
      • 一个非常简单的例子：下一张打出的牌的颜色应该和上一张的不同
    • 其他玩家推测规则，并打出自己认为符合规则的牌
    • 如果一个玩家成功打出一张牌，他可以试着猜出秘密规则是什么
  • 玩家数量
    • 3-8
    • 推荐数量 4-5
  • 牌堆
    • 两副标准扑克牌
    • 如果牌堆用尽，可以再洗一副牌，或者宣布游戏结束
  • 目标
    • 一局游戏由一轮或多轮组成
    • 每一轮中，玩家轮流担任庄家（决定秘密规则），庄家在其轮次中除了指定规则之外，只负责发牌，不参与出牌；庄家轮次的得分由其他玩家的分数来决定
    • 出牌以主线（mainline）和支线（sideline）方式摆放
      • 符合秘密规则的牌可以沿着主线放置，否则需要放在支线
  • 秘密规则
    • 如果第一次玩，建议从简单秘密规则示例中挑选一条
    • 一些关于制定秘密规则的提示
      • 规则应该允许在任何给定时间都能打出多张牌（但不能太多）
        • 即应该让至少一个玩家能够出牌，但最好无法让所有玩家都能出牌
          • 下一张牌必须大 1 —— 可能太难了
          • 下一张牌必须是不同花色 —— 可能太简单
      • 避免例外或者通用牌的出现
        • 比如人头牌总是对的
        • Ace 既可以是大牌也可以是小牌
      • 记住：无论是什么规则，它总比你想象中更难（被猜出）
  • 局前准备
    • 庄家写下秘密规则
    • 向每位玩家发 12 张牌
    • 把牌堆中的最上一张牌翻出作为主线的第一张牌
    • 游戏开始前，庄家可以选择是否提示规则，并决定出牌顺序
  • 局中
    • 出牌
      • 轮到玩家出一张牌
      • 庄家判定结果正确与否
        • 如果正确，放入主线
        • 如果错误，放入支线
      • 打错牌的玩家需要从牌堆中重新抽牌，使得手牌数量不变
      • 打对牌的玩家不用抓牌，手牌数量 - 1
    • 宣布不出牌
      • 玩家可以选择无牌可出
      • 由此，该玩家需要向所有玩家展示手牌
      • 庄家需要判定该玩家是否真的无牌可出
        • 如果玩家错了——有牌可出，那么庄家会从正确牌中选出一张放入主线，玩家保留手牌，然后从牌堆中抽一张
        • 如果玩家是对的——无牌可出
          • 如果玩家手中只有一张牌，该牌被放入牌堆，该轮结束
          • 如果玩家手中不止一张牌，所有牌被放入牌堆底部，庄家为其重新发牌（但总数少一张）
    • 猜测规则
      • 每当玩家成功出牌（放入主线）或是成功不出牌（确实无牌可出），就可以猜测规则
      • 猜测是公开进行的
      • 庄家会判断玩家的猜测是否正确
        • 如果正确，回合结束
        • 如果错误，游戏继续
  • 计分
    • 如果玩家正确猜出规则或一名玩家出完所有手牌，该回合结束
    • 玩家初始分数是 12 分
      • 正确猜出规则 +6 分
      • 出完所有手牌的玩家 +3 分
      • 手中每剩下一张牌 -1 分
    • 庄家得分与该轮场上得分最高的玩家分数相同
    • 然而
      • 如果不是要参加比赛，不用太在意得分
      • 重要的是享受有趣的游戏，让玩家能够猜出规则
        • 比如，如果玩到一半，庄家意识到可能规则太难了猜不出，他可以开始给出提示
          • 理论上讲这是犯规的，并且会增加庄家得分
          • 但没关系，这将使得游戏更有趣
        • 玩家应该可以随意讨论规则
  • 结束游戏
    • 游戏应该持续到每个玩家都成为过一次庄家为止
    • 也可以提前结束，那么就直接统计累加得分（在 [[New Eleusis]] 的规则中，如果没有当过庄家的人可以 +10 分）
  • 简单秘密规则示例（以下规则都指的是主线牌）
    • 下一张牌的颜色需要相反（即上一张是黑色，下一张就需要是红色）
    • 需要以黑红梅方的顺序出牌
    • 需要遵循三红、三黑、三红、三黑…的循环模式出牌
    • 如果当前是偶数，下一张需要是奇数；如果当前是奇数，下一张需要是偶数（J 是 11，Q 是 12，K 是 13）
    • 如果最后一张牌属于 1-7，需要出 8-K 中的牌；如果最后一张是 8-K 中的牌，需要出 1-7 中的牌
    • 下一张需要比上一张大 1/2/3（比如比 K-13 大 1 的牌是 1）
  • 困难秘密规则示例
    • 如果最后一张是奇数，则需要出红牌；如果最后一张是偶数，则需要出黑牌
    • 下一张牌必须与上一张的花色/数字相同
    • 如果当前是黑色，打出一张数字 <= 当前牌的牌；如果当前是红色，打出一张数字 >= 当前牌的牌

Components required

  • 两副标准扑克牌
  • 游戏规则

Deliverable

  • None

Suggested Process

  1. 玩游戏
    • 玩完后讨论每条规则，分析为什么规则太难/太简单/难度正好
  2. 写下学到的东西
    • 如果在玩的过程中学到了什么，写下来并与没有玩的人分享
