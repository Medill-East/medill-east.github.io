什么是 PKM

PKM 是 Personal Knowledge Management 的缩写，即个人知识管理

这是一种将脑内分散知识结构化的方法

一些常见的方法有：

  • 文件夹系统（比如 Windows 系统的文件夹）
  • 标签系统（比如 Mac 系统原生的标签系统）
  • 笔记（无论是实体或是电子化笔记）
  • 脑图（无论是实体或是电子化脑图）

个人的理解是，即使你可能对 PKM 这个概念并不熟悉，但其实日常生活中你对脑中知识进行结构化的每一次尝试，都可以被视作在践行 PKM

为什么关心 PKM

总的来说，PKM 可以在下列方面有所帮助：

  • 帮助你让生活更有条理
  • 优化你的学习效率
  • 帮助你迸发灵感

与个人而言，PKM 在以下方面帮助了我：

  • 学术（本科阶段）

    • Folder (with CmdMarkdown)
      • 最开始使用的是基于文件夹结构的 Markdown 工具 CmdMarkdown
        • 但是文件夹结构需要面临多份拷贝问题，即一次更改可能无法立马同步到其副本中
    • Mindmap (with XMind)
      • 并且辅以 XMind 来制作脑图
        • 但 XMind 文件过大之后使用不易
        • 不便于建立联系（搜索、获取、引用）
    • 由此便需要寻找新的工具

  • 不止于学术（硕士-至今）

    • Zettelkasten (with Roam Research)
      • 本科毕业后那段时间开始寻找新的笔记工具，也希望不只是学术产出，希望也能对生活的方方面面进行记录
        • 感觉双链笔记+卡片笔记法在那时（2020 年夏天）从 Roam Research 火了起来
        • 但因为 Roam Research 的以下原因，让我开始寻找别的工具
          • 付费比较高
          • 完全摒弃了文件夹结构，不太适应
          • 迁移不够方便
          • 基本都是网页编辑，编辑体验不够好（可能受到网速影响）
    • Zettelkasten (with Obsidian)
      • 也是在那段时间接触到了卢曼的卡片笔记法

使用 Obsidian 践行卢曼卡片法的好处有：

  • 易于输入
    • 因为支持 markdown 语法，所以很方便
  • 易于存储和迁移
    • 文件都是本地存储，可以自选云存储方案
  • 容易获取
    • 有方便和强大的搜索插件
  • 体验友好
    • 可以使用插件和主题来进行自定义
    • 好的心情能够更好地进行产出，《设计心理学》中对此进行过阐述

如何践行 PKM - 以卡片笔记法为例

Zettelkasten 是德文的“卡片盒”

  • 是什么
    • 这是 Niklas Luhmann 所创建的一种 PKM 方法
    • 他一生创建了 90000+ 卡片，发表了 58 本书，以及上百篇论文
  • 他的卡片盒笔记法中采用了双向链接对知识进行链接
    • 只有一份拷贝，所以不再担心需要同步更改的问题
    • 并且对相关的知识进行了链接（用线将相关卡片串在一起，并且将相关卡片放在相同盒子中）

  • 如何使用卡片笔记法
    • 我是从《[[How to take smart notes]]》（中文版 《卡片笔记写作法》）学习的
    • 其中包含了
      • [[fleeting notes]]
      • [[literature notes]]
      • [[permanent notes]]
      • [[SlipBox]]

一个笔记在建立的时候只需要根据这样的流程：

  • 是否有特殊的目的
    • 是
      • 插入独特标识（如时间戳 2024-0501-1206）
      • 插入模板
    • 否
      • 直接使用 unique note creator 插件创建
  • 是否是凭空得到的灵感
    • 是
      • 创建为 [[fleeting notes]]
    • 否
      • 那么应该是受到了材料的启发
        • 创建为 [[literature notes]]

  • 使用 PKM 进行输入
    • 结构
      • 获取灵感
      • 立马记录
        • 创建 [[fleeting notes]]
      • 逐步转化
        • [[fleeting notes]] -> [[literature notes]]
          • 添加来源
          • 添加引用格式（比如 zotero 的标识）
        • [[literature notes]] -> [[permanent notes]]
          • 让其能够自圆其说（每个笔记都可以视为一张卡片，那么每张卡片上的内容应该能顺畅地被理解，并且语意通顺，这样就可以在撰写文章时直接引用多张卡片内容，只需进行排列及书写连接词）
          • 添加相关的引用链接（父子关系/支持关系/反对关系等）
        • [[permanent notes]] -> [[SlipBox]]
          • 将永久笔记放入卡片盒中，方便索引

      • 日常实践
        • 对于新的笔记，都加上 “ToBeArchived” 的标签
        • 这样就可以通过 Dataview 将这些笔记都筛选出来
        • 尝试着每天都处理一些
          • 但因为输入速度总是要快于输出的，所以要做到这点确实非常不容易

  • 使用 PKM 进行输出
    • 构建联系
      • 使用双链 [[]] 来链接页面
      • 使用预览语法来链接内容 ![[title#section^paragraph]]
    • 输出文章
      • 从 slipbox 出发
        • 关注于 [[permanent notes]]
      • 不再是从零开始，而是从高质量的永久笔记开始输出

一些心得

让工作流顺畅起来！

  • 在 Obsidian 内
    • 善用插件！
      • 创建笔记
        • 可以使用 templates & Unique note creator
      • 创建双链
        • 最开始可以使用个人写的 Content Linker 来快速为笔记中的内容创建双链
        • 后期建议使用 Various Complements 来实现快速补全以提高输入效率
      • 搜索
        • Ominisearch 插件可以快速搜索库中所有内容
      • 其他有用插件
        • Dataview - 基于条件筛选内容
        • Easy Typing - 输入补全和增强（比如数字后面自动添加空格）
        • Enhancing Mindmap - 在 Ob 中可以快速创建脑图
        • Floating toc - 浮动目录
    • （使用主题和插件）让工作环境变得舒心！
      • 个人用的是 Blue Topaz 主题和 Style Settings 插件

  • 在 Ob 之外
    • 云端存储方案
      • 个人使用的是 Onedrive
    • 文献管理
      • 使用 Zotero（它也有方便的浏览器插件可以一键存入文献库中）
    • 图床（可以将图片以链接形式存储，减小库的大小，也便于博客使用）
      • PicGo（一键上传截图） + Github（图床）

总结

  • 保持思考！
  • 保持写作（输入+输出）！
  • 保持迭代（工作流）！
  • 所有的这些都只是工具，重要的是输出！
